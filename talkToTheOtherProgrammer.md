### 🚀 新增任务：编译器优化功能

#### 功能需求
1. **优化级别支持**:
   - `-O0` - 无优化（默认）
   - `-O1` - 部分优化
   - `-O2` - 高度优化
   - `-O3` - 极度优化（最高级）

2. **命令行接口**:
   ```bash
   python -m nova.repl -O0  # 无优化
   python -m nova.repl -O1  # 部分优化
   python -m nova.repl -O2  # 高度优化
   python -m nova.repl -O3  # 极度优化
   ```

3. **尾递归优化**:
   - 自动检测尾递归
   - 将尾递归转换为迭代
   - 提高性能，避免栈溢出

**任务 1: 实现优化级别系统**
1. **CLI 参数解析**
   - 在 `src/nova/cli/main.py` 中添加 `-O0`、`-O1`、`-O2`、`-O3` 参数

2. **优化器架构**
   - 在 `src/nova/compiler/optimizer/passes.py` 中实现不同级别的优化
   - 创建优化级别枚举：`OptimizationLevel` (O0, O1, O2, O3)

3. **优化实现**
   - **O0 (无优化)**:
     - 不进行任何优化
     - 保持原始代码结构
   
   - **O1 (部分优化)**:
     - 常量折叠
     - 死代码消除
     - 简单的内联
   
   - **O2 (高度优化)**:
     - 包含 O1 的所有优化
     - 循环优化
     - 表达式简化
     - 公共子表达式消除
   
   - **O3 (极度优化)**:
     - 包含 O2 的所有优化
     - 激进的内联
     - 循环展开
     - 更激进的优化

**任务 2: 实现尾递归优化**
1. **尾递归检测**
   - 在 AST 中识别尾递归调用
   - 检查递归调用是否在函数末尾

2. **尾递归转换**
   - 将尾递归转换为循环
   - 使用累加器模式
   - 避免栈增长

3. **示例转换**:
   ```nova
   // 转换前（尾递归）
   func factorial(n: int): int {
       if n <= 1 {
           return 1;
       }
       return n * factorial(n - 1);
   }
   
   // 转换后（迭代）
   func factorial(n: int): int {
       var result = 1;
       while n > 1 {
           result = result * n;
           n = n - 1;
       }
       return result;
   }
   ```
