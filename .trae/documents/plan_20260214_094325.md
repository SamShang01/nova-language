# Nova IDLE Shell 行为改进计划

## 目标

使 Nova IDLE 的 Shell 行为更像 Python IDLE，包括：

* 只读历史记录区域（不能编辑提示符之前的文本）

* 光标限制（只能在当前输入行中移动）

* 完整的文本编辑功能（backspace、delete、光标移动等）

## 实施步骤

### 1. 改进 `show_prompt` 方法

* 设置一个标记 `input_start`，标记输入的开始位置

* 确保标记正确定位在提示符 `>>> `  之后

### 2. 添加文本编辑限制

* 绑定 `<Key>` 事件，阻止在 `input_start` 标记之前进行编辑

* 绑定 `<Button-1>` 事件，阻止光标移动到 `input_start` 标记之前

* 绑定 `<MouseWheel>` 事件（可选），确保滚动正常

### 3. 实现完整的文本编辑功能

* 确保 Backspace、Delete、Home、End、左/右箭头等键正常工作

* 确保只在当前输入区域内工作

### 4. 改进 `on_shell_return` 方法

* 正确获取用户输入（从 `input_start` 到结尾）

* 执行后确保新的提示符正确显示

* 更新 `input_start` 标记位置

### 5. 改进 `replace_current_input` 方法

* 确保只替换当前输入区域的内容

* 保持 `input_start` 标记位置正确

### 6. 测试验证

* 测试不能编辑历史记录

* 测试光标不能移动到历史记录区域

* 测试在当前输入行中的完整编辑功能

* 测试命令历史记录功能（上/下箭头）
