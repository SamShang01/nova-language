Introduction to Collections

Collections are data structures that allow you to store and manipulate multiple values. Nova provides several built-in collection types, including arrays, lists, dictionaries, and sets.

Arrays

Arrays are fixed-size collections of elements of the same type. They are created using square brackets:

// Create an array of integers let numbers: array[int, 5] = [1, 2, 3, 4, 5]; // Access elements by index print(numbers[0]); // Output: 1 print(numbers[4]); // Output: 5 // Modify elements numbers[2] = 10; print(numbers); // Output: [1, 2, 10, 4, 5] // Get array length print(len(numbers)); // Output: 5

Note

Arrays in Nova have a fixed size that is determined at compile time. If you need a dynamic-size collection, use a list instead.

Lists

Lists are dynamic-size collections of elements of the same type. They are created using square brackets, similar to arrays:

// Create a list of strings let fruits: list[string] = ["apple", "banana", "cherry"]; // Access elements by index print(fruits[0]); // Output: apple // Add elements fruits.append("orange"); print(fruits); // Output: ["apple", "banana", "cherry", "orange"] // Insert elements fruits.insert(1, "grape"); print(fruits); // Output: ["apple", "grape", "banana", "cherry", "orange"] // Remove elements fruits.remove("banana"); print(fruits); // Output: ["apple", "grape", "cherry", "orange"] // Pop elements let last_fruit = fruits.pop(); print(last_fruit); // Output: orange print(fruits); // Output: ["apple", "grape", "cherry"] // Get list length print(len(fruits)); // Output: 3

List Methods

  • append(item) - Adds an item to the end of the list
  • insert(index, item) - Inserts an item at the specified index
  • remove(item) - Removes the first occurrence of the item
  • pop(index) - Removes and returns the item at the specified index (default is last item)
  • clear() - Removes all items from the list
  • index(item) - Returns the index of the first occurrence of the item
  • count(item) - Returns the number of occurrences of the item
  • sort() - Sorts the list in place
  • reverse() - Reverses the list in place

Dictionaries

Dictionaries are collections of key-value pairs. They are created using curly braces:

// Create a dictionary let person: dict[string, any] = { "name": "Alice", "age": 30, "city": "New York" }; // Access values by key print(person["name"]); // Output: Alice // Modify values person["age"] = 31; print(person); // Output: {"name": "Alice", "age": 31, "city": "New York"} // Add new key-value pairs person["job"] = "Engineer"; print(person); // Output: {"name": "Alice", "age": 31, "city": "New York", "job": "Engineer"} // Remove key-value pairs person.remove("city"); print(person); // Output: {"name": "Alice", "age": 31, "job": "Engineer"} // Check if a key exists print("name" in person); // Output: true print("city" in person); // Output: false // Get dictionary keys let keys = person.keys(); print(keys); // Output: ["name", "age", "job"] // Get dictionary values let values = person.values(); print(values); // Output: ["Alice", 31, "Engineer"] // Get dictionary items (key-value pairs) let items = person.items(); print(items); // Output: [("name", "Alice"), ("age", 31), ("job", "Engineer")]

Dictionary Methods

  • remove(key) - Removes the key-value pair with the specified key
  • keys() - Returns a list of all keys
  • values() - Returns a list of all values
  • items() - Returns a list of key-value pairs
  • clear() - Removes all key-value pairs
  • get(key, default) - Returns the value for the key, or the default if the key doesn't exist

Sets

Sets are unordered collections of unique elements. They are created using the set function:

// Create a set let numbers: set[int] = set([1, 2, 3, 4, 5]); // Add elements numbers.add(6); print(numbers); // Output: {1, 2, 3, 4, 5, 6} // Add a duplicate (no effect) numbers.add(3); print(numbers); // Output: {1, 2, 3, 4, 5, 6} // Remove elements numbers.remove(2); print(numbers); // Output: {1, 3, 4, 5, 6} // Check if an element exists print(1 in numbers); // Output: true print(2 in numbers); // Output: false // Get set size print(len(numbers)); // Output: 5 // Set operations let set1: set[int] = set([1, 2, 3, 4]); let set2: set[int] = set([3, 4, 5, 6]); // Union let union = set1.union(set2); print(union); // Output: {1, 2, 3, 4, 5, 6} // Intersection let intersection = set1.intersection(set2); print(intersection); // Output: {3, 4} // Difference let difference = set1.difference(set2); print(difference); // Output: {1, 2}

Set Methods

  • add(item) - Adds an item to the set
  • remove(item) - Removes an item from the set
  • clear() - Removes all items from the set
  • union(other) - Returns a new set with elements from both sets
  • intersection(other) - Returns a new set with elements common to both sets
  • difference(other) - Returns a new set with elements in the first set but not in the second
  • is_subset(other) - Returns true if all elements of the set are in the other set
  • is_superset(other) - Returns true if all elements of the other set are in the set

Tuples

Tuples are immutable ordered collections of elements of different types. They are created using parentheses:

// Create a tuple let person: (string, int, string) = ("Alice", 30, "New York"); // Access elements by index print(person[0]); // Output: Alice print(person[1]); // Output: 30 print(person[2]); // Output: New York // Tuple unpacking let (name, age, city) = person; print(name); // Output: Alice print(age); // Output: 30 print(city); // Output: New York // Tuples are immutable // person[1] = 31; // Error: Cannot modify tuple

Note

Tuples are immutable, meaning their elements cannot be changed after creation. Use tuples when you need a fixed collection of values that won't change.

Working with Collections

Iterating Over Collections

You can use loops to iterate over collections:

// Iterate over a list let fruits = ["apple", "banana", "cherry"]; for (let fruit in fruits) { print(fruit); } // Output: apple, banana, cherry // Iterate over a dictionary let person = {"name": "Alice", "age": 30}; for (let key, value in person) { print(key + ": " + value); } // Output: name: Alice, age: 30 // Iterate over a set let numbers = set([1, 2, 3]); for (let number in numbers) { print(number); } // Output: 1, 2, 3

List Comprehensions

List comprehensions provide a concise way to create lists:

// Create a list of squares let squares = [x * x for x in [1, 2, 3, 4, 5]]; print(squares); // Output: [1, 4, 9, 16, 25] // Create a list of even numbers let even_numbers = [x for x in [1, 2, 3, 4, 5, 6] if x % 2 == 0]; print(even_numbers); // Output: [2, 4, 6] // Create a list of tuples let pairs = [(x, y) for x in [1, 2] for y in [3, 4]]; print(pairs); // Output: [(1, 3), (1, 4), (2, 3), (2, 4)]

Collection Operations

You can perform various operations on collections:

// Concatenate lists let list1 = [1, 2, 3]; let list2 = [4, 5, 6]; let combined = list1 + list2; print(combined); // Output: [1, 2, 3, 4, 5, 6] // Repeat lists let repeated = [1, 2] * 3; print(repeated); // Output: [1, 2, 1, 2, 1, 2] // Check membership let fruits = ["apple", "banana", "cherry"]; print("apple" in fruits); // Output: true print("orange" in fruits); // Output: false // Get min and max let numbers = [5, 2, 8, 1, 9]; print(min(numbers)); // Output: 1 print(max(numbers)); // Output: 9 // Sum elements print(sum(numbers)); // Output: 25

Best Practices

  • Choose the right collection type: Use arrays for fixed-size collections, lists for dynamic-size collections, dictionaries for key-value pairs, sets for unique elements, and tuples for immutable collections.
  • Use type annotations: Specify types for collections to improve code clarity and catch errors early.
  • Prefer list comprehensions: Use list comprehensions for concise and readable list creation.
  • Use appropriate methods: Familiarize yourself with the methods available for each collection type to write more efficient code.
  • Consider performance: Be aware of the performance characteristics of different collection operations.

Common Mistakes

  • Using arrays for dynamic collections: Arrays have fixed size, so use lists for collections that need to grow or shrink.
  • Modifying tuples: Tuples are immutable, so you can't change their elements.
  • Using the wrong key type for dictionaries: Dictionary keys must be hashable (e.g., strings, numbers, tuples).
  • Forgetting that sets are unordered: Sets don't preserve insertion order, so don't rely on the order of elements.
  • Using inefficient operations: For example, using in on a list is O(n), while using it on a set is O(1).
