Introduction to Classes

Classes are the fundamental building blocks of object-oriented programming (OOP) in Nova. They allow you to define blueprints for creating objects with specific properties and behaviors.

Defining a Class

You can define a class using the class keyword:

class Person { let name: string; let age: int; def __init__(self, name: string, age: int) { self.name = name; self.age = age; } def greet(self) { print("Hello, my name is " + self.name); } }

In this example:

  • class Person defines a new class named Person
  • let name: string; and let age: int; define class properties
  • def __init__(self, name: string, age: int) is the constructor method
  • def greet(self) is a method of the class

Creating Objects

To create an object (instance) of a class, use the new keyword:

// Create an instance of Person let person = new Person("Alice", 30); // Access properties print(person.name); // Output: Alice print(person.age); // Output: 30 // Call methods person.greet(); // Output: Hello, my name is Alice

Constructors

Constructors are special methods that are called when an object is created. In Nova, the constructor method is named __init__:

class Car { let make: string; let model: string; let year: int; // Constructor with parameters def __init__(self, make: string, model: string, year: int) { self.make = make; self.model = model; self.year = year; } // Constructor with default values def __init__(self, make: string, model: string) { self.make = make; self.model = model; self.year = 2023; } // Default constructor def __init__(self) { self.make = "Unknown"; self.model = "Unknown"; self.year = 2023; } } // Using different constructors let car1 = new Car("Toyota", "Camry", 2022); let car2 = new Car("Honda", "Civic"); let car3 = new Car();

Note

Nova supports method overloading, including constructors. This means you can define multiple constructors with different parameter lists.

Methods

Methods are functions defined within a class. They can access and modify the class's properties:

class Circle { let radius: float; def __init__(self, radius: float) { self.radius = radius; } def area(self) -> float { return 3.14159 * self.radius * self.radius; } def circumference(self) -> float { return 2 * 3.14159 * self.radius; } def set_radius(self, new_radius: float) { if (new_radius > 0) { self.radius = new_radius; } else { print("Radius must be positive"); } } } let circle = new Circle(5.0); print(circle.area()); // Output: 78.53975 print(circle.circumference()); // Output: 31.4159 circle.set_radius(10.0); print(circle.area()); // Output: 314.159

Inheritance

Inheritance allows you to create a new class that inherits properties and methods from an existing class. The new class is called a subclass, and the existing class is called a superclass:

// Superclass class Animal { let name: string; def __init__(self, name: string) { self.name = name; } def make_sound(self) { print("Some generic sound"); } } // Subclass class Dog extends Animal { let breed: string; def __init__(self, name: string, breed: string) { // Call the superclass constructor super(name); self.breed = breed; } // Override the make_sound method def make_sound(self) { print("Woof! Woof!"); } // Additional method def fetch(self) { print(self.name + " is fetching"); } } let animal = new Animal("Generic Animal"); animal.make_sound(); // Output: Some generic sound let dog = new Dog("Buddy", "Golden Retriever"); dog.make_sound(); // Output: Woof! Woof! dog.fetch(); // Output: Buddy is fetching print(dog.name); // Output: Buddy (inherited from Animal)

Note

When overriding a method, you can use the super keyword to call the superclass's version of the method.

Access Modifiers

Nova provides access modifiers to control the visibility of class members:

  • public - Accessible from anywhere (default)
  • private - Only accessible within the class
  • protected - Accessible within the class and its subclasses
class BankAccount { private let balance: float; public let account_number: string; protected let owner: string; def __init__(self, account_number: string, owner: string) { self.account_number = account_number; self.owner = owner; self.balance = 0.0; } public def deposit(self, amount: float) { if (amount > 0) { self.balance += amount; } } public def withdraw(self, amount: float) { if (amount > 0 && amount <= self.balance) { self.balance -= amount; return true; } return false; } public def get_balance(self) -> float { return self.balance; } } class SavingsAccount extends BankAccount { private let interest_rate: float; def __init__(self, account_number: string, owner: string, interest_rate: float) { super(account_number, owner); self.interest_rate = interest_rate; } public def add_interest() { let current_balance = self.get_balance(); let interest = current_balance * self.interest_rate; self.deposit(interest); print("Interest added: " + interest); } public def get_owner() -> string { return self.owner; // Accessible because owner is protected } } let account = new BankAccount("12345", "Alice"); account.deposit(1000); print(account.get_balance()); // Output: 1000 // print(account.balance); // Error: balance is private let savings = new SavingsAccount("67890", "Bob", 0.05); savings.deposit(5000); savings.add_interest(); // Output: Interest added: 250 print(savings.get_balance()); // Output: 5250 print(savings.get_owner()); // Output: Bob

Static Members

Static members belong to the class itself, not to individual instances. You can define static properties and methods using the static keyword:

class MathUtils { static let PI: float = 3.14159; static def add(a: float, b: float) -> float { return a + b; } static def multiply(a: float, b: float) -> float { return a * b; } } // Access static properties and methods without creating an instance print(MathUtils.PI); // Output: 3.14159 print(MathUtils.add(5, 3)); // Output: 8 print(MathUtils.multiply(4, 6)); // Output: 24

Abstract Classes and Methods

An abstract class is a class that cannot be instantiated and is meant to be subclassed. Abstract methods are methods that are declared but not implemented in the abstract class, and must be implemented by subclasses:

abstract class Shape { abstract def area() -> float; abstract def perimeter() -> float; def describe() { print("This is a shape"); } } class Rectangle extends Shape { let width: float; let height: float; def __init__(self, width: float, height: float) { self.width = width; self.height = height; } def area() -> float { return self.width * self.height; } def perimeter() -> float { return 2 * (self.width + self.height); } } class Circle extends Shape { let radius: float; def __init__(self, radius: float) { self.radius = radius; } def area() -> float { return 3.14159 * self.radius * self.radius; } def perimeter() -> float { return 2 * 3.14159 * self.radius; } } // let shape = new Shape(); // Error: Cannot instantiate abstract class let rectangle = new Rectangle(5, 3); print(rectangle.area()); // Output: 15 print(rectangle.perimeter()); // Output: 16 rectangle.describe(); // Output: This is a shape let circle = new Circle(4); print(circle.area()); // Output: 50.26544 print(circle.perimeter()); // Output: 25.13272 circle.describe(); // Output: This is a shape

Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common superclass. This enables you to write code that works with any subclass of a given type:

class Animal { def make_sound() { print("Some generic sound"); } } class Dog extends Animal { def make_sound() { print("Woof! Woof!"); } } class Cat extends Animal { def make_sound() { print("Meow! Meow!"); } } // Polymorphic function function let animals_make_sound(animals: list[Animal]) { for (let animal in animals) { animal.make_sound(); // Calls the appropriate method based on the actual type } } let animals: list[Animal] = [ new Dog(), new Cat(), new Animal() ]; animals_make_sound(animals); // Output: // Woof! Woof! // Meow! Meow! // Some generic sound

Best Practices

  • Encapsulation: Use access modifiers to protect class members and expose only what's necessary.
  • Inheritance: Use inheritance to reuse code and create a hierarchy of related classes.
  • Polymorphism: Use polymorphism to write flexible code that works with different types.
  • Abstract classes: Use abstract classes to define common interfaces for subclasses.
  • Meaningful names: Use clear, descriptive names for classes, properties, and methods.
  • Single responsibility: Each class should have a single, well-defined responsibility.

Common Mistakes

  • Forgetting the self parameter: All instance methods require a self parameter as the first argument.
  • Accessing private members from outside the class: Private members can only be accessed within the class.
  • Not calling the superclass constructor: When extending a class, make sure to call the superclass constructor using super().
  • Overriding methods without maintaining the same signature: When overriding a method, keep the same parameter list and return type.
  • Creating overly complex class hierarchies: Keep inheritance hierarchies simple and focused.
