Introduction to Variables

Variables are used to store data in a program. In Nova, variables are declared using the let keyword, followed by the variable name and an optional initial value.

Declaring Variables

In Nova, you can declare variables in several ways:

Basic Declaration

// Declare a variable without initial value let x; // Declare a variable with initial value let y = 10; let name = "Nova"; let is_active = true;

Type Annotations

Nova supports optional type annotations to specify the type of a variable:

// With type annotations let x: int = 10; let name: string = "Nova"; let is_active: bool = true;

Type Inference

Nova has type inference, which means it can automatically determine the type of a variable based on its initial value. Type annotations are optional but can make your code more readable and help catch errors early.

Variable Types

Nova supports several built-in data types:

Primitive Types

  • int: Integer numbers (e.g., 10, -5, 0)
  • float: Floating-point numbers (e.g., 3.14, -0.5, 2.0)
  • bool: Boolean values (true or false)
  • string: Text strings (e.g., "Hello", "Nova")
  • char: Single characters (e.g., 'a', '1', '@')

Example Usage

let age: int = 25; let pi: float = 3.14159; let is_student: bool = true; let message: string = "Hello, Nova!"; let initial: char = 'N';

Variable Scope

Variable scope determines where a variable is accessible in your code. Nova has two types of variable scope:

Local Scope

Variables declared inside a function or block are only accessible within that function or block:

func main() { let local_var = "I'm local"; print(local_var); // This works } // print(local_var); // This would cause an error - local_var is not accessible here

Global Scope

Variables declared outside of any function are accessible throughout the program:

let global_var = "I'm global"; func main() { print(global_var); // This works } func another_function() { print(global_var); // This also works }

Best Practice

Use local variables whenever possible to avoid naming conflicts and make your code more modular. Global variables should be used sparingly.

Variable Mutability

By default, variables in Nova are mutable, meaning you can change their values after declaration:

let count = 0; print(count); // Output: 0 count = 1; print(count); // Output: 1 count = count + 1; print(count); // Output: 2

Constants

To create a constant (a value that cannot be changed), use the const keyword:

const PI = 3.14159; // PI = 3.14; // This would cause an error - constants cannot be modified

Naming Conventions

Nova follows these naming conventions for variables:

  • Variable names: Use camelCase (e.g., firstName, age)
  • Constants: Use UPPERCASE with underscores (e.g., MAX_VALUE, PI)
  • Function names: Use camelCase (e.g., calculateTotal, printMessage)
  • Class names: Use PascalCase (e.g., Person, Car)

Valid Variable Names

  • Must start with a letter or underscore
  • Can contain letters, numbers, and underscores
  • Are case-sensitive (myVar and myvar are different)
  • Cannot be reserved keywords (e.g., let, const, func)

Working with Variables

Here are some common operations with variables:

Arithmetic Operations

let a = 10; let b = 5; let sum = a + b; // 15 let difference = a - b; // 5 let product = a * b; // 50 let quotient = a / b; // 2 let remainder = a % b; // 0

String Operations

let firstName = "John"; let lastName = "Doe"; let fullName = firstName + " " + lastName; // "John Doe" let greeting = "Hello, " + fullName + "!"; // "Hello, John Doe!"

Boolean Operations

let isTrue = true; let isFalse = false; let andResult = isTrue && isFalse; // false let orResult = isTrue || isFalse; // true let notResult = !isTrue; // false

Type Conversion

You can convert between different types using type conversion functions:

// Convert to string let num = 10; let numStr = String(num); // "10" // Convert to int let str = "42"; let strInt = Int(str); // 42 // Convert to float let intNum = 5; let floatNum = Float(intNum); // 5.0

Type Safety

Nova is a statically-typed language, so type conversions must be explicit. You cannot directly assign a value of one type to a variable of another type without conversion.

Common Mistakes

  • Using undeclared variables: Always declare variables before using them.
  • Type mismatches: Make sure you're assigning values of the correct type to variables.
  • Shadowing variables: Be careful not to declare a variable with the same name as an existing variable in the same scope.
  • Modifying constants: Remember that constants cannot be changed after declaration.
