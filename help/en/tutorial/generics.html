Using generics for type-safe code in Nova

Introduction to Generics

Generics allow you to write code that works with multiple types without sacrificing type safety. They enable you to create reusable components that can operate on different types while maintaining compile-time type checking.

Generic Functions

You can define generic functions by specifying type parameters in angle brackets (<T>):

// Generic function def printArray(arr: Array) { for (let item in arr) { print(item); } } // Usage with different types let numbers = [1, 2, 3, 4, 5]; let strings = ["hello", "world", "nova"]; printArray(numbers); // Output: 1, 2, 3, 4, 5 printArray(strings); // Output: hello, world, nova

Multiple Type Parameters

You can define functions with multiple type parameters:

// Multiple type parameters def pair(first: T, second: U) -> Array { return [first, second]; } // Usage let result1 = pair(1, "hello"); print(result1); // Output: [1, "hello"] let result2 = pair(true, 3.14); print(result2); // Output: [true, 3.14]

Generic Classes

You can define generic classes by specifying type parameters in angle brackets:

// Generic class class Box { private let value: T; def __init__(self, value: T) { self.value = value; } def getValue() -> T { return self.value; } def setValue(value: T) { self.value = value; } } // Usage with different types let intBox = Box(42); print(intBox.getValue()); // Output: 42 intBox.setValue(100); print(intBox.getValue()); // Output: 100 let stringBox = Box("Hello"); print(stringBox.getValue()); // Output: Hello stringBox.setValue("World"); print(stringBox.getValue()); // Output: World

Multiple Type Parameters in Classes

Classes can also have multiple type parameters:

// Multiple type parameters in class class Pair { let first: T; let second: U; def __init__(self, first: T, second: U) { self.first = first; self.second = second; } def getFirst() -> T { return self.first; } def getSecond() -> U { return self.second; } } // Usage let pair1 = Pair(1, "one"); print(pair1.getFirst()); // Output: 1 print(pair1.getSecond()); // Output: one let pair2 = Pair("test", true); print(pair2.getFirst()); // Output: test print(pair2.getSecond()); // Output: true

Generic Interfaces and Traits

You can define generic interfaces and traits:

// Generic trait trait Container { def add(item: T); def remove() -> T; def isEmpty() -> bool; } // Implementing generic trait class Stack implements Container { private let items: Array = []; def add(item: T) { self.items.push(item); } def remove() -> T { if (self.isEmpty()) { throw new Error("Stack is empty"); } return self.items.pop(); } def isEmpty() -> bool { return self.items.length == 0; } } // Usage let stack = Stack(); stack.add(1); stack.add(2); stack.add(3); print(stack.remove()); // Output: 3 print(stack.remove()); // Output: 2 print(stack.remove()); // Output: 1 print(stack.isEmpty()); // Output: true

Type Constraints

You can add constraints to type parameters to ensure they meet certain requirements:

// Type constraints trait Comparable { def compareTo(other: any) -> int; } class Person implements Comparable { let name: string; let age: int; def __init__(self, name: string, age: int) { self.name = name; self.age = age; } def compareTo(other: any) -> int { if (other is Person) { return self.age - other.age; } return 0; } } // Generic function with constraint def findMax(items: Array) -> T { if (items.length == 0) { throw new Error("Array is empty"); } let max = items[0]; for (let item in items) { if (item.compareTo(max) > 0) { max = item; } } return max; } // Usage let people = [ Person("John", 30), Person("Alice", 25), Person("Bob", 35) ]; let oldest = findMax(people); print(oldest.name); // Output: Bob

Default Type Parameters

You can specify default types for type parameters:

// Default type parameters def defaultValue() -> T { if (T is string) { return "default" as T; } elif (T is int) { return 0 as T; } elif (T is bool) { return false as T; } return null as T; } // Usage print(defaultValue()); // Output: default print(defaultValue()); // Output: 0 print(defaultValue()); // Output: false print(defaultValue()); // Output: default (uses string default)

Generic Methods in Non-Generic Classes

You can define generic methods in non-generic classes:

// Generic methods in non-generic class class Utils { def static printList(list: Array) { for (let item in list) { print(item); } } def static map(list: Array, mapper: (T) -> U) -> Array { let result: Array = []; for (let item in list) { result.push(mapper(item)); } return result; } } // Usage let numbers = [1, 2, 3, 4, 5]; Utils.printList(numbers); // Output: 1, 2, 3, 4, 5 let doubled = Utils.map(numbers, (x) => x * 2); Utils.printList(doubled); // Output: 2, 4, 6, 8, 10 let strings = Utils.map(numbers, (x) => "Number: " + x); Utils.printList(strings); // Output: Number: 1, Number: 2, Number: 3, Number: 4, Number: 5

Wildcard Types

You can use wildcard types to represent unknown types:

// Wildcard types class Box { private let value: T; def __init__(self, value: T) { self.value = value; } def getValue() -> T { return self.value; } } // Using wildcard type def printBoxContent(box: Box) { print(box.getValue()); } // Usage let intBox = Box(42); let stringBox = Box("Hello"); printBoxContent(intBox); // Output: 42 printBoxContent(stringBox); // Output: Hello

Type Erasure

In Nova, generics are implemented using type erasure, which means that type parameters are removed at runtime. This allows for efficient code generation while maintaining compile-time type safety.

Note

Because of type erasure, you cannot use type parameters in runtime type checks like typeof T or is T inside generic functions or classes.

Best Practices

  • Use descriptive type parameter names: Use meaningful names like T, U, V for simple cases, or more descriptive names like ElementType for complex cases.
  • Add type constraints when necessary: Use constraints to ensure type parameters meet specific requirements.
  • Keep generics simple: Don't overcomplicate your code with too many type parameters.
  • Use generics for collection types: Generics are particularly useful for collection classes like lists, stacks, and queues.
  • Document generic code: Add comments to explain how generic types are used.

Common Mistakes

  • Forgetting to specify type parameters: Always specify type parameters when creating instances of generic classes.
  • Using type parameters in runtime checks: Remember that type parameters are erased at runtime.
  • Overusing generics: Don't use generics when a simple function or class would suffice.
  • Ignoring type constraints: Make sure to implement all required methods when using constrained type parameters.
  • Mixing up type parameters: Be careful with the order of multiple type parameters.
