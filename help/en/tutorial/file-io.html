Introduction to File I/O

File I/O (Input/Output) operations allow you to read from and write to files. Nova provides a comprehensive set of functions and methods for working with files.

Opening and Closing Files

To work with a file, you first need to open it using the open function. After you're done with the file, you should close it using the close method.

// Opening and closing a file let file = open("example.txt", "r"); try { // File operations here print("File opened successfully"); } finally { file.close(); print("File closed"); }

File Modes

When opening a file, you can specify the mode:

  • "r" - Read mode (default)
  • "w" - Write mode (overwrites existing file)
  • "a" - Append mode (appends to existing file)
  • "r+" - Read and write mode
  • "w+" - Write and read mode (overwrites existing file)
  • "a+" - Append and read mode
  • "b" - Binary mode (e.g., "rb", "wb")

Reading Files

Reading Entire File

You can read the entire content of a file using the read method:

// Reading entire file using (let file = open("example.txt", "r")) { let content = file.read(); print(content); }

Reading Line by Line

You can read a file line by line using the readline method or by iterating over the file:

// Reading line by line with readline using (let file = open("example.txt", "r")) { let line = file.readline(); while (line != null) { print(line.strip()); // Strip newline character line = file.readline(); } } // Reading line by line with for loop using (let file = open("example.txt", "r")) { for (let line in file) { print(line.strip()); } }

Reading All Lines

You can read all lines of a file into a list using the readlines method:

// Reading all lines using (let file = open("example.txt", "r")) { let lines = file.readlines(); for (let line in lines) { print(line.strip()); } }

Writing Files

Writing to a File

You can write to a file using the write method:

// Writing to a file using (let file = open("output.txt", "w")) { file.write("Hello, world!\n"); file.write("This is a test.\n"); } // Output: Creates output.txt with the written content

Writing Multiple Lines

You can write multiple lines at once using the writelines method:

// Writing multiple lines let lines = ["Line 1\n", "Line 2\n", "Line 3\n"]; using (let file = open("output.txt", "w")) { file.writelines(lines); } // Output: Creates output.txt with the three lines

Appending to Files

To append content to an existing file, open it in append mode ("a"):

// Appending to a file using (let file = open("output.txt", "a")) { file.write("Appended line 1\n"); file.write("Appended line 2\n"); } // Output: Adds the new lines to the end of output.txt

Working with Binary Files

To work with binary files, open them in binary mode by adding "b" to the mode:

// Reading a binary file using (let file = open("image.jpg", "rb")) { let data = file.read(); print("Read " + len(data) + " bytes"); } // Writing a binary file let data: bytes = [0x48, 0x65, 0x6c, 0x6c, 0x6f]; // "Hello" in ASCII using (let file = open("binary.bin", "wb")) { file.write(data); } // Output: Creates binary.bin with the byte data

File Position

You can get and set the current position in a file using the tell and seek methods:

// File position operations using (let file = open("example.txt", "r")) { // Get current position let position = file.tell(); print("Initial position: " + position); // Read some data let data = file.read(10); print("Read: " + data); // Get new position position = file.tell(); print("Position after reading: " + position); // Seek to beginning file.seek(0); print("Position after seeking to 0: " + file.tell()); // Read again from beginning data = file.read(10); print("Read again: " + data); }

Seek Modes

The seek method takes an optional second parameter to specify the seek mode:

  • 0 - Seek from the beginning of the file (default)
  • 1 - Seek from the current position
  • 2 - Seek from the end of the file

File System Operations

Nova provides various functions for working with the file system:

// File system operations import os; // Check if a file exists if (os.path.exists("example.txt")) { print("File exists"); } else { print("File does not exist"); } // Get file size if (os.path.exists("example.txt")) { let size = os.path.getsize("example.txt"); print("File size: " + size + " bytes"); } // Check if a path is a file or directory if (os.path.isfile("example.txt")) { print("It's a file"); } elif (os.path.isdir("example")) { print("It's a directory"); } // List files in a directory let files = os.listdir("."); print("Files in current directory:"); for (let file in files) { print(file); } // Create a directory os.makedirs("new_directory", true); // true means create intermediate directories // Remove a file if (os.path.exists("temp.txt")) { os.remove("temp.txt"); print("File removed"); } // Remove a directory if (os.path.exists("empty_directory")) { os.rmdir("empty_directory"); print("Directory removed"); }

Error Handling for File Operations

File operations can throw exceptions, so it's important to handle them properly:

// Error handling for file operations try { using (let file = open("nonexistent.txt", "r")) { let content = file.read(); print(content); } } catch (FileNotFoundError e) { print("Error: File not found - " + e.message); } catch (IOError e) { print("Error: I/O error - " + e.message); } catch (Error e) { print("Error: " + e.message); }

Using Statement

The using statement automatically closes resources when they go out of scope, which is useful for file operations:

// Using statement for file operations // This is equivalent to try-finally using (let file = open("example.txt", "r")) { let content = file.read(); print(content); } // File is automatically closed here

Note

The using statement works with any resource that has a close method. It ensures that the resource is closed even if an exception occurs.

Best Practices

  • Always close files: Use the using statement or try-finally to ensure files are closed.
  • Handle exceptions: Catch and handle exceptions that may occur during file operations.
  • Use appropriate modes: Choose the correct file mode for your operation.
  • Check file existence: Verify that files exist before trying to read them.
  • Be mindful of file sizes: Reading large files all at once can consume a lot of memory.
  • Use binary mode for binary files: Always open binary files in binary mode.
  • Clean up resources: Remove temporary files when they're no longer needed.

Common Mistakes

  • Forgetting to close files: This can lead to resource leaks.
  • Using the wrong mode: For example, trying to write to a file opened in read mode.
  • Not handling exceptions: File operations can fail for various reasons.
  • Reading large files all at once: This can cause memory issues.
  • Not checking file existence: Trying to read a non-existent file will throw an exception.
  • Using text mode for binary files: This can corrupt binary data.
