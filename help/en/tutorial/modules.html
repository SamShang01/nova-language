Introduction to Modules

Modules are a way to organize code into separate files, making it easier to manage large codebases. Nova supports modules through a simple and intuitive system.

Creating a Module

To create a module, simply create a new .nova file. The filename (without the extension) becomes the module name.

// math.nova def add(a: float, b: float) -> float { return a + b; } def subtract(a: float, b: float) -> float { return a - b; } def multiply(a: float, b: float) -> float { return a * b; } def divide(a: float, b: float) -> float { if (b == 0) { throw new Error("Division by zero"); } return a / b; }

Importing Modules

You can import modules using the import keyword:

// main.nova import math; let result = math.add(5, 3); print("5 + 3 = " + result); // Output: 5 + 3 = 8 result = math.subtract(10, 4); print("10 - 4 = " + result); // Output: 10 - 4 = 6 result = math.multiply(6, 7); print("6 * 7 = " + result); // Output: 6 * 7 = 42 result = math.divide(20, 5); print("20 / 5 = " + result); // Output: 20 / 5 = 4

Importing Specific Functions

You can import specific functions from a module:

// main.nova import { add, multiply } from math; let sum = add(5, 3); print("5 + 3 = " + sum); // Output: 5 + 3 = 8 let product = multiply(6, 7); print("6 * 7 = " + product); // Output: 6 * 7 = 42 // subtract and divide are not available here

Importing with Aliases

You can import modules or functions with aliases:

// main.nova // Import module with alias import math as m; let result = m.add(5, 3); print("5 + 3 = " + result); // Output: 5 + 3 = 8 // Import specific functions with aliases import { add as sum, multiply as times } from math; let s = sum(5, 3); print("5 + 3 = " + s); // Output: 5 + 3 = 8 let t = times(6, 7); print("6 * 7 = " + t); // Output: 6 * 7 = 42

Exporting from Modules

By default, all functions, classes, and variables in a module are exported. However, you can use the export keyword to explicitly export items, and private to prevent items from being exported:

// utils.nova // This function is exported by default def public_function() { print("This is a public function"); } // This function is explicitly exported export def exported_function() { print("This is an exported function"); } // This function is not exported private def private_function() { print("This is a private function"); } // This variable is exported by default let public_variable = 42; // This variable is explicitly exported export let exported_variable = "Hello"; // This variable is not exported private let private_variable = "Secret";
// main.nova import utils; utils.public_function(); // Works utils.exported_function(); // Works // utils.private_function(); // Error: private_function is not exported print(utils.public_variable); // Works print(utils.exported_variable); // Works // print(utils.private_variable); // Error: private_variable is not exported

Module Structure

For larger projects, you can organize modules into directories. Each directory can have an index.nova file that serves as the entry point for the module.

Directory Structure Example

my_project/ ├── main.nova └── utils/ ├── index.nova ├── math.nova └── string.nova

Index File

The index.nova file can re-export items from other modules in the directory:

// utils/index.nova export * from math; export * from string;

Using the Module

// main.nova import utils; // Using functions from math.nova let result = utils.add(5, 3); print(result); // Using functions from string.nova let text = "hello world"; let capitalized = utils.capitalize(text); print(capitalized);

Built-in Modules

Nova provides several built-in modules that you can import:

  • os - Operating system utilities
  • sys - System-specific parameters and functions
  • math - Mathematical functions
  • random - Random number generation
  • datetime - Date and time functions
  • json - JSON parsing and serialization
  • io - Input/output operations
// Using built-in modules import os; import math; import random; // OS module print(os.getcwd()); // Get current working directory // Math module print(math.pi); // Output: 3.141592653589793 print(math.sqrt(16)); // Output: 4.0 // Random module print(random.randint(1, 10)); // Output: Random number between 1 and 10

Module Resolution

When you import a module, Nova looks for it in the following locations:

  1. The current directory
  2. Directories specified in the NOVA_PATH environment variable
  3. Standard library directories

Note

Module names are case-sensitive, so make sure to use the correct case when importing modules.

Circular Imports

Circular imports occur when two or more modules import each other. Nova handles circular imports gracefully, but it's generally best to avoid them to keep your code clean and maintainable.

Warning

Circular imports can lead to unexpected behavior, especially if modules try to use each other's functions during initialization.

Best Practices

  • Organize code logically: Group related functionality into the same module.
  • Use descriptive module names: Choose module names that reflect their purpose.
  • Keep modules small: Each module should have a single responsibility.
  • Use explicit exports: Use the export keyword to make it clear what's available to other modules.
  • Avoid circular imports: Refactor code to eliminate circular dependencies.
  • Use directory structures for large projects: Organize modules into directories with index files.
  • Document modules: Add comments to explain what a module does and how to use it.

Common Mistakes

  • Incorrect module name: Make sure the module name matches the filename.
  • Case sensitivity: Module names are case-sensitive, so Math and math are different.
  • Circular imports: Avoid circular dependencies between modules.
  • Forgetting to export: Make sure to export items that other modules need to use.
  • Importing from non-existent modules: Check that the module exists and is in the correct location.
