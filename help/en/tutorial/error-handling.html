Introduction to Error Handling

Error handling is an important aspect of programming that allows you to gracefully handle unexpected situations. Nova provides a comprehensive error handling system using try-catch blocks and exceptions.

Try-Catch Blocks

You can use try-catch blocks to catch and handle exceptions:

// Basic try-catch try { // Code that might throw an exception let result = 10 / 0; print(result); } catch (e) { // Code to handle the exception print("Error: " + e.message); } // Output: Error: Division by zero

Multiple Catch Blocks

You can use multiple catch blocks to handle different types of exceptions:

// Multiple catch blocks try { let array = [1, 2, 3]; print(array[5]); // Index out of bounds } catch (IndexError e) { print("Index error: " + e.message); } catch (Error e) { print("General error: " + e.message); } // Output: Index error: Index out of bounds

Finally Block

The finally block is executed regardless of whether an exception is thrown or not:

// Try-catch-finally try { print("Executing try block"); let result = 10 / 0; print(result); } catch (e) { print("Error: " + e.message); } finally { print("Executing finally block"); } // Output: // Executing try block // Error: Division by zero // Executing finally block

Throwing Exceptions

You can throw exceptions using the throw keyword:

// Throwing an exception def divide(a: float, b: float) -> float { if (b == 0) { throw new Error("Division by zero"); } return a / b; } try { let result = divide(10, 0); print(result); } catch (e) { print("Error: " + e.message); } // Output: Error: Division by zero

Custom Exceptions

You can create custom exception classes by extending the base Error class:

// Custom exception class ValidationError extends Error { def __init__(self, message: string) { super(message); self.name = "ValidationError"; } } def validate_age(age: int) { if (age < 0) { throw new ValidationError("Age cannot be negative"); } else if (age > 120) { throw new ValidationError("Age is too high"); } print("Age is valid: " + age); } try { validate_age(-5); } catch (ValidationError e) { print("Validation error: " + e.message); } catch (Error e) { print("General error: " + e.message); } // Output: Validation error: Age cannot be negative

Exception Types

Nova provides several built-in exception types:

  • Error - Base exception class
  • TypeError - Raised when an operation or function is applied to an object of inappropriate type
  • ValueError - Raised when a function receives an argument of the correct type but inappropriate value
  • IndexError - Raised when trying to access an index that is out of range
  • KeyError - Raised when trying to access a key that doesn't exist in a dictionary
  • ZeroDivisionError - Raised when dividing by zero
  • FileNotFoundError - Raised when trying to access a file that doesn't exist
  • IOError - Raised when an I/O operation fails
// Example of different exception types // TypeError try { let result = "5" + 5; // Trying to add string and number print(result); } catch (TypeError e) { print("Type error: " + e.message); } // IndexError try { let array = [1, 2, 3]; print(array[10]); // Index out of bounds } catch (IndexError e) { print("Index error: " + e.message); } // KeyError try { let person = {"name": "Alice"}; print(person["age"]); // Key doesn't exist } catch (KeyError e) { print("Key error: " + e.message); }

Error Handling Best Practices

Be Specific

Catch specific exception types rather than catching all exceptions:

// Good: Catch specific exception try { let result = 10 / 0; } catch (ZeroDivisionError e) { print("Cannot divide by zero"); } // Bad: Catch all exceptions try { let result = 10 / 0; } catch (e) { print("An error occurred"); }

Provide Meaningful Error Messages

Use descriptive error messages that help identify the problem:

// Good: Descriptive error message def calculate_discount(price: float, discount: float) -> float { if (discount < 0) { throw new ValueError("Discount cannot be negative"); } else if (discount > 100) { throw new ValueError("Discount cannot exceed 100%"); } return price * (1 - discount / 100); } // Bad: Vague error message def calculate_discount(price: float, discount: float) -> float { if (discount < 0 || discount > 100) { throw new ValueError("Invalid discount"); } return price * (1 - discount / 100); }

Use Finally for Cleanup

Use the finally block for cleanup operations that should always execute:

// Using finally for cleanup def read_file(filename: string) { let file = null; try { file = open(filename, "r"); // Read file content print("Reading file..."); // Simulate an error throw new Error("Simulated error"); } catch (e) { print("Error: " + e.message); } finally { if (file != null) { file.close(); print("File closed"); } } } read_file("example.txt"); // Output: // Reading file... // Error: Simulated error // File closed

Don't Swallow Exceptions

Don't catch exceptions without handling them properly:

// Bad: Swallowing exceptions try { let result = 10 / 0; } catch (e) { // Do nothing } // Good: Handling exceptions try { let result = 10 / 0; } catch (e) { print("Error: " + e.message); // Or rethrow the exception if you can't handle it // throw e; }

Error Propagation

Exceptions can be propagated up the call stack if they are not caught:

// Error propagation def divide(a: float, b: float) -> float { if (b == 0) { throw new Error("Division by zero"); } return a / b; } def calculate(a: float, b: float) -> float { // Exception is not caught here, so it propagates to the caller return divide(a, b); } try { let result = calculate(10, 0); print(result); } catch (e) { print("Error caught: " + e.message); } // Output: Error caught: Division by zero

Common Error Handling Patterns

Try-With-Resources

In Nova, you can use the using statement to automatically close resources:

// Using statement for resource management using (let file = open("example.txt", "r")) { // File is automatically closed when leaving this block let content = file.read(); print(content); } // File is closed here

Error Handling with Result Type

For functions that might fail, you can return a Result type instead of throwing exceptions:

// Using Result type def safe_divide(a: float, b: float) -> Result[float, string] { if (b == 0) { return Result.error("Division by zero"); } return Result.ok(a / b); } let result = safe_divide(10, 2); if (result.is_ok()) { print("Result: " + result.unwrap()); } else { print("Error: " + result.unwrap_err()); } // Output: Result: 5 let result2 = safe_divide(10, 0); if (result2.is_ok()) { print("Result: " + result2.unwrap()); } else { print("Error: " + result2.unwrap_err()); } // Output: Error: Division by zero

Debugging with Exceptions

Exceptions can help you debug your code by providing stack traces:

// Stack trace example def function_a() { function_b(); } def function_b() { function_c(); } def function_c() { throw new Error("Something went wrong"); } try { function_a(); } catch (e) { print("Error: " + e.message); print("Stack trace:"); print(e.stack); } // Output: // Error: Something went wrong // Stack trace: // at function_c (example.nova:10) // at function_b (example.nova:6) // at function_a (example.nova:2) // at main (example.nova:14)

Common Mistakes

  • Catching all exceptions: This can hide bugs and make debugging difficult.
  • Swallowing exceptions: Not handling exceptions properly can lead to unexpected behavior.
  • Throwing exceptions for normal control flow: Exceptions should be used for exceptional situations, not for normal program flow.
  • Not providing meaningful error messages: Vague error messages make it hard to identify and fix problems.
  • Not cleaning up resources: Forgetting to close files or release resources can lead to resource leaks.
