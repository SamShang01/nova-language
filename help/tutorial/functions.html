Introduction to Functions

Functions are reusable blocks of code that perform a specific task. They help organize code, improve readability, and reduce repetition. In Nova, functions are declared using the def keyword.

Defining Functions

To define a function in Nova, use the def keyword followed by the function name, parentheses containing any parameters, and a block of code enclosed in curly braces:

// Function definition def greet() { print("Hello, World!"); } // Function call greet(); // Output: Hello, World!

Function Parameters

Functions can accept parameters (also called arguments) to make them more flexible:

// Function with parameters def greet(name) { print("Hello, " + name + "!"); } // Function call with arguments greet("John"); // Output: Hello, John! greet("Nova"); // Output: Hello, Nova!

Type Annotations

You can add type annotations to function parameters for better type safety:

// Function with type annotations def add(a: int, b: int) { return a + b; } let result = add(5, 3); print(result); // Output: 8

Return Values

Functions can return values using the return keyword:

// Function with return value def add(a, b) { return a + b; } let sum = add(5, 3); print(sum); // Output: 8

Return Type Annotations

You can also specify the return type of a function:

// Function with return type annotation def add(a: int, b: int) -> int { return a + b; }

Default Parameters

Functions can have default values for parameters:

// Function with default parameter def greet(name, greeting = "Hello") { print(greeting + ", " + name + "!"); } // Call with both arguments greet("John", "Hi"); // Output: Hi, John! // Call with only required argument greet("John"); // Output: Hello, John!

Variable Arguments

Functions can accept a variable number of arguments using the * syntax:

// Function with variable arguments def sum_numbers(*numbers) { let total = 0; for (let number in numbers) { total += number; } return total; } let result1 = sum_numbers(1, 2, 3); print(result1); // Output: 6 let result2 = sum_numbers(1, 2, 3, 4, 5); print(result2); // Output: 15

Keyword Arguments

You can pass arguments by name, which makes the code more readable:

// Function with multiple parameters def describe_person(name, age, city) { print("Name: " + name); print("Age: " + String(age)); print("City: " + city); } // Call with keyword arguments describe_person(name: "John", age: 30, city: "New York");

Function Scope

Variables declared inside a function are local to that function and cannot be accessed outside:

func main() { let local_var = "I'm local"; print(local_var); // This works } // print(local_var); // This would cause an error

Accessing Global Variables

Functions can access global variables, but if you want to modify them, you need to use the global keyword:

let global_var = 10; func modify_global() { global global_var; global_var = 20; } print(global_var); // Output: 10 modify_global(); print(global_var); // Output: 20

Recursive Functions

Functions can call themselves, which is known as recursion:

// Recursive function to calculate factorial def factorial(n) { if (n <= 1) { return 1; } return n * factorial(n - 1); } let result = factorial(5); print(result); // Output: 120

Caution

Recursive functions can lead to stack overflow if not implemented carefully. Make sure to include a base case that stops the recursion.

Anonymous Functions

Nova supports anonymous functions (also called lambda functions) using the lambda keyword:

// Anonymous function let add = lambda a, b => a + b; let result = add(5, 3); print(result); // Output: 8

Using Anonymous Functions as Arguments

// Function that takes another function as argument def apply_operation(a, b, operation) { return operation(a, b); } // Pass anonymous function as argument let result = apply_operation(5, 3, lambda a, b => a + b); print(result); // Output: 8 let result2 = apply_operation(5, 3, lambda a, b => a * b); print(result2); // Output: 15

Function Composition

You can combine multiple functions to create new functions:

// Define helper functions def add_one(x) { return x + 1; } def multiply_by_two(x) { return x * 2; } // Compose functions def compose(f, g) { return lambda x => f(g(x)); } // Create new function let add_one_then_double = compose(multiply_by_two, add_one); let result = add_one_then_double(5); print(result); // Output: 12 (5 + 1 = 6, 6 * 2 = 12)

Built-in Functions

Nova provides several built-in functions that you can use:

  • print(...args): Prints values to the console
  • len(collection): Returns the length of a collection
  • type(value): Returns the type of a value
  • int(value): Converts a value to an integer
  • float(value): Converts a value to a float
  • string(value): Converts a value to a string
  • bool(value): Converts a value to a boolean
// Using built-in functions print("Hello, Nova!"); // Output: Hello, Nova! print(len([1, 2, 3])); // Output: 3 print(type(42)); // Output: int print(int("42")); // Output: 42

Function Best Practices

  • Single Responsibility: Each function should do one thing and do it well.
  • Descriptive Names: Use clear, descriptive names that explain what the function does.
  • Proper Parameter Names: Use meaningful names for parameters.
  • Type Annotations: Use type annotations to improve code readability and catch errors.
  • Documentation: Add comments to explain complex functions.
  • Return Early: Return early from functions to reduce nesting.

Common Mistakes

  • Forgetting Parentheses: Remember to add parentheses when calling a function.
  • Incorrect Number of Arguments: Make sure to pass the correct number of arguments.
  • Variable Shadowing: Be careful not to shadow variables from outer scopes.
  • Infinite Recursion: Always include a base case in recursive functions.
