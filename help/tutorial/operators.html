Introduction to Operators

Operators are symbols that perform operations on values. Nova supports a wide range of operators, including arithmetic, comparison, logical, and assignment operators.

Arithmetic Operators

Arithmetic operators perform mathematical operations:

Operator Description Example Result
+ Addition 5 + 3 8
- Subtraction 10 - 4 6
* Multiplication 6 * 7 42
/ Division 20 / 5 4
% Modulo (remainder) 10 % 3 1
** Exponentiation 2 ** 3 8
// Integer division 10 // 3 3
// Arithmetic operations let a = 10; let b = 3; print(a + b); // Output: 13 print(a - b); // Output: 7 print(a * b); // Output: 30 print(a / b); // Output: 3.3333333333333335 print(a % b); // Output: 1 print(a ** b); // Output: 1000 print(a // b); // Output: 3

Comparison Operators

Comparison operators compare two values and return a boolean:

Operator Description Example Result
== Equal to 5 == 5 true
!= Not equal to 5 != 3 true
> Greater than 5 > 3 true
< Less than 5 < 3 false
>= Greater than or equal to 5 >= 5 true
<= Less than or equal to 5 <= 3 false
=== Strict equal (value and type) 5 === 5 true
!== Strict not equal 5 !== "5" true
// Comparison operations let a = 5; let b = 3; let c = 5; let d = "5"; print(a == b); // Output: false print(a == c); // Output: true print(a != b); // Output: true print(a > b); // Output: true print(a < b); // Output: false print(a >= c); // Output: true print(a <= b); // Output: false print(a === d); // Output: false (different types) print(a !== d); // Output: true (different types)

Logical Operators

Logical operators perform logical operations on boolean values:

Operator Description Example Result
&& Logical AND true && false false
|| Logical OR true || false true
! Logical NOT !true false
// Logical operations let x = true; let y = false; print(x && y); // Output: false print(x || y); // Output: true print(!x); // Output: false print(!y); // Output: true // Short-circuit evaluation print(false && someNonExistentFunction()); // Doesn't call the function print(true || someNonExistentFunction()); // Doesn't call the function

Note

Nova uses short-circuit evaluation for logical operators. This means that if the result can be determined from the first operand, the second operand is not evaluated.

Assignment Operators

Assignment operators assign values to variables:

Operator Description Example Equivalent
= Simple assignment x = 5 x = 5
+= Addition assignment x += 3 x = x + 3
-= Subtraction assignment x -= 3 x = x - 3
*= Multiplication assignment x *= 3 x = x * 3
/= Division assignment x /= 3 x = x / 3
%= Modulo assignment x %= 3 x = x % 3
**= Exponentiation assignment x **= 3 x = x ** 3
//= Integer division assignment x //= 3 x = x // 3
// Assignment operations let x = 10; print(x); // Output: 10 x += 5; print(x); // Output: 15 x -= 3; print(x); // Output: 12 x *= 2; print(x); // Output: 24 x /= 4; print(x); // Output: 6 x %= 5; print(x); // Output: 1 x **= 3; print(x); // Output: 1 x = 10; x //= 3; print(x); // Output: 3

Unary Operators

Unary operators operate on a single operand:

Operator Description Example Result
+ Unary plus +5 5
- Unary minus -5 -5
! Logical NOT !true false
++ Increment x++ x before increment
-- Decrement x-- x before decrement
// Unary operations let x = 5; print(+x); // Output: 5 print(-x); // Output: -5 let y = true; print(!y); // Output: false let z = 10; print(z++); // Output: 10 (returns z before increment) print(z); // Output: 11 print(--z); // Output: 10 (returns z before decrement) print(z); // Output: 10

Bitwise Operators

Bitwise operators operate on the binary representation of numbers:

Operator Description Example Result (binary) Result (decimal)
& Bitwise AND 5 & 3 0001 1
| Bitwise OR 5 | 3 0111 7
^ Bitwise XOR 5 ^ 3 0110 6
~ Bitwise NOT ~5 ... -6 (two's complement)
<< Left shift 5 << 1 1010 10
>> Right shift 5 >> 1 0010 2
>>> Unsigned right shift -5 >>> 1 ... 2147483645 (unsigned)
// Bitwise operations let a = 5; // Binary: 0101 let b = 3; // Binary: 0011 print(a & b); // Output: 1 (0001) print(a | b); // Output: 7 (0111) print(a ^ b); // Output: 6 (0110) print(~a); // Output: -6 (two's complement) print(a << 1); // Output: 10 (1010) print(a >> 1); // Output: 2 (0010)

String Operators

String operators work with string values:

Operator Description Example Result
+ String concatenation "Hello" + " " + "World" "Hello World"
+= String concatenation assignment s += " World" s becomes "Hello World"
* String repetition "Hello" * 3 "HelloHelloHello"
// String operations let s1 = "Hello"; let s2 = "World"; print(s1 + " " + s2); // Output: Hello World let s3 = "Hello"; s3 += " World"; print(s3); // Output: Hello World print("Hello" * 3); // Output: HelloHelloHello

Array and Object Operators

Operators for working with arrays and objects:

Operator Description Example Result
[] Array/object access arr[0], obj["key"] Element at index or key
in Check if element/key exists 5 in arr, "key" in obj Boolean
... Spread operator [...arr], {...obj} New array/object
// Array and object operations // Array operations let arr = [1, 2, 3, 4, 5]; print(arr[0]); // Output: 1 print(3 in arr); // Output: true print(6 in arr); // Output: false // Spread operator let arr2 = [...arr, 6, 7]; print(arr2); // Output: [1, 2, 3, 4, 5, 6, 7] // Object operations let obj = { name: "John", age: 30 }; print(obj["name"]); // Output: John print(obj.name); // Output: John print("age" in obj); // Output: true print("address" in obj); // Output: false // Spread operator let obj2 = { ...obj, address: "123 Main St" }; print(obj2); // Output: { name: "John", age: 30, address: "123 Main St" }

Operator Precedence

Operators have different precedence levels, which determine the order in which operations are performed:

Precedence Operators Associativity
1 () (grouping) N/A
2 ** Right
3 ++, --, !, ~, + (unary), - (unary) Right
4 *, /, %, // Left
5 +, - Left
6 <<, >>, >>> Left
7 <, <=, >, >= Left
8 ==, !=, ===, !== Left
9 & Left
10 ^ Left
11 | Left
12 && Left
13 || Left
14 =, +=, -=, *=, /=, etc. Right
15 , (comma) Left
// Operator precedence examples // Without parentheses: 10 + 5 * 2 = 10 + 10 = 20 print(10 + 5 * 2); // Output: 20 // With parentheses: (10 + 5) * 2 = 15 * 2 = 30 print((10 + 5) * 2); // Output: 30 // Unary minus has higher precedence than multiplication print(-5 * 2); // Output: -10 // Exponentiation is right-associative: 2 ** 3 ** 2 = 2 ** 9 = 512 print(2 ** 3 ** 2); // Output: 512

Best Practices

  • Use parentheses for clarity: Even if operator precedence is correct, use parentheses to make code more readable.
  • Be consistent: Use consistent spacing around operators to improve readability.
  • Understand precedence: Familiarize yourself with operator precedence to avoid unexpected results.
  • Use compound assignments: Use +=, -=, etc. for cleaner code.
  • Be careful with floating-point operations: Remember that floating-point operations can have precision issues.
  • Use string interpolation: For complex string concatenation, consider using string interpolation instead of the + operator.

Common Mistakes

  • Confusing assignment and equality: Using = instead of == in conditions.
  • Ignoring operator precedence: Not using parentheses when needed, leading to unexpected results.
  • Mixing data types: Using arithmetic operators on incompatible data types.
  • Forgetting short-circuit evaluation: Assuming that both operands of logical operators will always be evaluated.
  • Using bitwise operators on non-integers: Bitwise operators should only be used on integer values.
